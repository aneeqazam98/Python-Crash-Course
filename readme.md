# 🐍 Python Crash Course – Structured Learning Repository

This repository is a structured, class-based Python learning system designed for beginners who want to move from zero knowledge to practical programming understanding in a clean and organized way.

The goal is not just to learn Python, but to build a **developer-style learning workflow** using real project structure and version control.

---

# 🎯 Purpose of This Repository

This project is designed to:

* Build strong programming fundamentals in Python
* Provide structured learning through classes
* Separate theory, practice, and notes clearly
* Train real-world development habits (folder structure, documentation, practice files)
* Serve as a personal reference system for revision and future projects

---

# 📁 Project Structure

The repository follows a **class-based modular structure**:

```
Python-Crash-Course/
│
├── Class-1/
│   ├── class-1-notes.md        → Complete notes and explanations
│   ├── practice-1.ipynb   → Interactive practice notebook
│
├── Class-2/
│   ├── class-2-notes.md        → Complete notes and explanations
│   ├── practice-2.ipynb   → Interactive practice notebook
│
├── Class-3/
│   ├── class-3-notes.md        → Complete notes and explanations
│   ├── practice-3.ipynb   → Interactive practice notebook
│
└── README.md            → Overview of full course (this file)
```

---

# 🧠 Explanation of Structure

## 📌 Class-Based System

Each class represents a **learning module** covering specific Python concepts.

This approach helps in:

* Breaking down learning into manageable parts
* Avoiding confusion from mixed topics
* Building step-by-step understanding

---

## 📘 README.md (Inside Each Class)

Each class folder contains a `README.md` file which includes:

* Theoretical explanations of concepts
* Code examples
* Expected outputs
* Beginner-friendly breakdowns

👉 This is your **main study material**

---

## 🧪 practice.ipynb (Jupyter Notebook)

* Used for interactive learning
* Combines code + explanation
* Helps in testing concepts step-by-step

👉 Best for experimentation and learning visually

---

## 💻 practice.py (Python File)

* Pure code practice file
* Used for running programs in terminal or VS Code
* Helps build real coding habit

👉 Best for execution and real-world practice

---

# ⚙ Why This Structure Matters

Most beginners make the mistake of mixing everything in one file. This repository avoids that problem by using a **separation of concerns approach**:

* Notes → README.md
* Practice → .py file
* Interactive learning → .ipynb

This structure is similar to how professional developers organize real projects.

---

# 🚀 How to Use This Repository

### Step 1

Open a class folder (e.g. Class-1)

### Step 2

Start reading `README.md` for theory

### Step 3

Practice in:

* `practice.py` (terminal execution)
* `practice.ipynb` (interactive learning)

### Step 4

Move to next class after completing practice
